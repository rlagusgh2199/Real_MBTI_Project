# Real MBTI 프로젝트 Constitution (프로젝트 원칙) - v2

## 1. 프로젝트 목적

- 설문이 아닌 **실제 행동 데이터(카카오톡 대화, 유튜브 시청 기록, SNS 활동 로그 등)**를 기반으로
  사용자의 성향을 분석하는 "Real MBTI" 시스템을 만든다.
- 생성형 AI(LLM)를 활용하여 사용자가 이해하기 쉬운 **자연어 리포트**를 자동으로 생성한다.
- 이 프로젝트는 수업 과제이지만, 단순 데모가 아니라 **실제 서비스처럼 시연 가능한 프로토타입**을 목표로 한다.

## 2. 핵심 원칙

1. **실제 행동 데이터 기반 입력**

   - 사용자가 직접 텍스트를 타이핑하는 방식이 아니라,
     실제 사용 기록을 내보내기(export)한 파일을 입력으로 사용한다.
     - 예: 카카오톡 대화 내보내기 `.txt`
     - 예: 유튜브 시청 기록 내보내기 파일
     - 예: SNS(인스타/트위터 등) 활동 내보내기 파일
   - 모든 분석은 사용자의 **로컬 PC 환경에서만 수행**하며,
     데이터는 서버로 업로드하지 않는다.

2. **투명한 분석 과정**

   - "어떤 로그 → 어떤 특징(Features) → 어떤 MBTI 점수"인지
     분석 파이프라인을 코드와 문서로 명확히 남긴다.
   - 데이터 소스별로 어떤 패턴을 봤는지(예: 카톡 대화 패턴 vs 유튜브 시청 패턴)를
     설명할 수 있어야 한다.

3. **설명 가능한 결과와 신뢰도 표시**

   - MBTI 결과와 함께, 각 축(E/I, S/N, T/F, J/P)에 대해
     "어떤 행동 패턴 때문에 이런 점수가 나왔는지"를 설명한다.
   - 분석에 사용된 데이터의 양·기간·출처에 따라
     **"신뢰도(Confidence)" 또는 "참고용 지표"**를 함께 보여준다.
     - 예: 데이터가 너무 적을 경우 "참고용" 정도로 표시
     - 예: 특정 기간/소스에만 치우친 경우 그 한계를 설명

4. **재현 가능한 규칙 기반 점수 계산**

   - 입력 데이터가 같으면 항상 같은 MBTI 점수가 나오도록 한다.
   - MBTI 점수 계산은 **규칙/수식 기반 알고리즘**으로 구현한다.
   - LLM은 이 결과를 설명·요약하는 역할에 집중시키고,
     점수 자체는 LLM에게 맡기지 않는다.

5. **신빙성(Validity)을 의식한 설계**

   - 이 시스템의 결과는 "심리검사 대체"가 아니라
     **행동 데이터 기반 참고 지표**라는 점을 명시한다.
   - 가능하다면, 사용자의 실제 MBTI(설문 결과)와
     Real MBTI 결과를 비교해서 어느 정도 일치하는지 검증할 수 있는 구조를 남긴다.
     - 예: 몇 명의 테스트 사용자에 대해 설문 MBTI vs Real MBTI를 비교·기록
     - 예: 축별로 자주 어긋나는 부분은 규칙/가중치를 조정

## 3. 사용자 경험(UX) 원칙

1. **파일 업로드 중심 입력 흐름**

   - 웹 UI에서 "카카오톡 내보내기 파일 업로드", "유튜브 기록 파일 업로드",
     "SNS 활동 파일 업로드" 같은 입력 방식을 제공한다.
   - 사용자가 별도의 텍스트를 직접 작성하지 않아도,
     **실제 생활에서 생성된 로그 데이터만으로 분석이 가능**해야 한다.

2. **한눈에 보이는 결과 + 신뢰도 표기**

   - MBTI 4축 점수(E/I, S/N, T/F, J/P)를 그래프나 표로 시각화한다.
   - "데이터 양/기간/출처 기반 신뢰도 표시"를 함께 보여준다.
     - 예: "최근 3개월 카카오톡 + 유튜브 기록 기반, 신뢰도: 중간"
   - LLM 리포트는 "요약 → 축별 설명 → 생활/관계/일에서의 특징 → 한계/주의사항" 순서로 보여준다.

3. **데이터 편향/한계를 사용자에게 알려주기**

   - 특정 소스에만 데이터가 몰려 있으면(예: 유튜브만, 카톡만),
     "어떤 상황에서의 성향이 더 반영된 결과인지" 설명한다.
   - 사용자가 다른 소스를 추가로 업로드하면 결과가 어떻게 바뀔 수 있는지 안내한다.

## 4. 기술적인 가이드라인

- (기존과 동일: Python, Local LLM(Ollama), 모듈 구조: data_loader / feature_extractor / mbti_scorer / llm_reporter / app 등)
- 단, `data_loader`는 **카카오톡/유튜브/SNS 각각의 내보내기 포맷을 파싱하는 역할**을 맡는다.

## 5. 범위(Scope)

- 이 프로젝트는 **수업 과제이면서, 실제 서비스에 가까운 프로토타입**을 목표로 한다.
- 로컬 환경에서 동작하는 개인용 분석 도구를 기본으로 하되,
  시연이 가능한 웹 UI를 제공하는 것을 1차 목표로 한다.
- 상용 서비스 수준의 법적/보안 검토는 범위 밖이지만,
  개인정보는 로컬에만 머무르도록 설계한다.
